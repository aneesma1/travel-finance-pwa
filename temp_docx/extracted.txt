Points for New Developer 9.1  Things That Will Bite You node --check passes but the browser still throws â€” Template literals with nested braces pass syntax check but can confuse brace-counting tools. The real test is the browser console, not node --check. Service worker serves stale files â€” If you change JS files but the SW cache name does not change, users get old code. Always change the CACHE_NAME in sw.js when deploying. The stamp script (/tmp/stamp_version.py) does this automatically. localSave lives in sync-manager.js, NOT drive.js â€” drive.js has writeData(). These are different. localSave() writes to IndexedDB first, then queues Drive write. writeData() goes directly to Drive. Import path depth matters â€” screens/ files need ../../../shared/, not ../../shared/. Wrong paths cause silent module failures with no error in some browsers. Two listeners on same button = two modals â€” Multiple addEventListeners on the same element accumulate. Always check if an element was already bound before adding listeners. Use a data attribute flag or remove-then-add pattern. getCachedFinanceData() can return null on first open â€” Drive fetch is async. The transactions screen must not assume data exists immediately. OAuth is Implicit Flow, not PKCE â€” Do not change this. PKCE requires a client secret. The client secret cannot be stored safely in a public repo. isAdmin() reads a module-level variable set by setCurrentRole() â€” If Settings renders before setCurrentRole() is called, isAdmin() returns false and the Manage Access button is hidden. 10.
